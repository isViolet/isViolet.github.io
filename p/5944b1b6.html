使用 MySQL 的图形界面管理工具 Navicat for MySQL 连接 Mysql 数据库时提示错误：Can't connect to MySQL server (10060)

# 【出现该问题可能的原因：】

  1. 网络不通；
  2. 服务未启动；
  3. 防火墙未关闭；
  4. 服务器上防火墙端口未开放；
  5. 端口未被监听；
  6. 权限不足。

# 【解决办法：】

  1. 自行百度配置网络

  2. 启动服务

    #在安装目录下
[root@VM_0_5_centos sbin]# pwd
/software/mysql-5.5.32/sbin
[root@VM_0_5_centos sbin]# ./mysqld start
Starting MySQL SUCCESS!

  3. 关闭防火墙，不会的点这里 Linux 常用命令大全（非常全！！！）

  4. 在自己购买的服务器上将 MySQL 使用的 3306 端口开放，如果是在本地搭建的虚拟机，记得一定要满足第 1 条，将网络接通外网。

  5. 添加监听的端口：

    添加需要监听的端口
/sbin/iptables -I INPUT -p tcp --dport 3306 -j ACCEPT

保存设置/etc/init.d/iptables save
查看状态/etc/init.d/iptables status

  6. 权限不足 权限问题，参考：navicat 连接不上 Linux 服务器上的 MySQL
